# 开发文档命名规范

> 适用范围：`/Users/a1/work/work_space` 下记录**开发任务**的文档。
> 一篇开发文档 = 一项开发任务，记录其目标、方案与全过程进展。
> 命名与解析文档区别开：解析文档按序号编号，开发文档一律**以日期开头**。

---

## 1. 文件命名

统一格式：

```
<日期>-<当日序号>-<任务主题>.md
```

| 部分 | 规则 | 示例 |
|------|------|------|
| 日期 | 八位 `YYYYMMDD`，为任务**创建当天** | `20260701` |
| 当日序号 | 两位数字，从 `01` 起，同一天内递增 | `01` `02` `03` |
| 分隔符 | 各部分之间用连字符 `-` | |
| 任务主题 | 简洁中文短语，**语义清晰、见名知意**，不含空格；避免 `开发文档1` 这类无意义命名 | `登录接口重构` `数据管线优化` |
| 扩展名 | 统一 `.md` | |

示例：

```
20260701-01-登录接口重构.md
20260701-02-数据管线优化.md
20260702-01-推理缓存接入.md
```

> 同一天的多篇开发文档用 `-01` `-02` 递增区分；跨天则日期变化，序号重新从 `01` 起。

---

## 2. 元信息头

每篇文件**开头第一行起**，用 HTML 注释块记录管理信息（不影响 Markdown 渲染）：

```markdown
<!--
name: <任务主题，与文件名一致>
creator: Li Cheng
created: YYYY-MM-DD
modified: YYYY-MM-DD
related_issues: <关联的问题记录文档，无则留空或写 -->
```

| 字段 | 含义 | 规则 |
|------|------|------|
| `name` | 任务主题 | 与文件名主题一致 |
| `creator` | 创建人 | 本目录默认 **Li Cheng**，他人创建改为实际姓名 |
| `created` | 创建时间 | 首次创建日期，创建后不再改动 |
| `modified` | 最后修改时间 | 每次实质性更新后改为当日 |
| `related_issues` | 关联问题 | 填对应的问题记录文档名（可多个），见第 5 节 |

---

## 3. 开发状态（必填）

采用**最简三阶段**状态机，正文首个标题下用醒目一行标注：

```markdown
**状态**：开发中
```

| 状态 | 含义 |
|------|------|
| 未启动 | 已建文档、明确目标，但尚未动工 |
| 开发中 | 正在实现，进展记录持续追加 |
| 已完成 | 任务完成并验证通过 |

> 状态只进不退：`未启动 → 开发中 → 已完成`。状态变更时同步更新 `modified`。

---

## 4. 正文结构（推荐，保持精简）

按以下小节组织，**没有内容的小节可省略**，避免冗长：

```markdown
# <日期>-<序号> · <任务主题>

**状态**：开发中

## 1. 目标与背景
做什么、为什么做（1~3 句说清）。

## 2. 影响范围
涉及的代码仓 / 模块 / 关键文件。

## 3. 实现方案
关键设计与实现要点（列要点即可，不写长篇）。

## 4. 进展记录
按日期倒序或正序追加，天然形成时间线：
- 2026-07-01：完成初版接口，待联调。
- 2026-07-02：联调通过，补充单测。

## 5. 关联问题
开发中发现的问题，链接到对应问题记录文档（见第 5 节）。

## 6. 验证与结果
如何验证（测试、运行结果），完成后填写。
```

原则：**能一屏看懂**。写清目标、方案、进展即可，不追求面面俱到。

---

## 5. 与问题记录文档的对应

开发过程中发现的问题，**另建问题记录文档**（见 `问题记录文档命名规范.md`），并双向关联：

- 开发文档：元信息头 `related_issues` + 第 5 节列出关联问题文档名。
- 问题文档：元信息头 `related_dev` 指回本开发文档名。

示例对应：

```
开发文档  20260701-01-登录接口重构.md
              └─ related_issues: 20260701-02-登录超时问题.md
问题文档  20260701-02-登录超时问题.md
              └─ related_dev: 20260701-01-登录接口重构.md
```

---

## 6. 新建开发文档速查

1. 命名：`<今天日期>-<当日序号>-<主题>.md`（序号当日从 `01` 递增）。
2. 顶部补元信息头，`creator` 默认 `Li Cheng`，`created`/`modified` 填当日。
3. 正文首行标 `**状态**：未启动`，写明目标与背景。
4. 动工后改 `开发中`，持续追加进展记录。
5. 遇到问题→建问题记录文档并双向关联。
6. 完成并验证后改 `已完成`。
